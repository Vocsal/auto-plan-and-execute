# STATUS 收敛标记协议

本协议被 `plan-review` 和 `execution-review` 两个角色严格遵守。它是 auto-flow.sh 判断"是否继续迭代"的唯一信号。

## 标记格式

审查类报告**必须**以下面两种格式之一结束（独占一行，无前后缀，无 markdown 修饰）：

```
STATUS: PASS
```

或

```
STATUS: NEEDS_REVISION
```

## 判定规则

依据"问题严重等级"映射：

| 问题分布 | STATUS |
|---|---|
| 存在任意 P0 问题 | NEEDS_REVISION |
| 存在任意 P1 问题 | NEEDS_REVISION |
| 仅 P2 / P3 问题 | PASS |
| 无任何问题 | PASS |

## 问题严重等级

- **P0（阻断）**：根本性错误、关键需求缺失、技术上不可行、安全漏洞、测试不过、与需求严重不符 — 必须修复
- **P1（严重）**：关键设计缺陷、重要场景遗漏、风险未识别、明显代码质量问题 — 强烈建议修复
- **P2（一般）**：可改进的细节、不影响主流程的优化建议、次要场景缺测试 — 可考虑修复
- **P3（提示）**：风格、措辞、可选优化 — 可不修复

## 避免连续否决

如果当前是该阶段的最后一轮（依据脚本传入的"当前轮 / 最大轮"信息），可以更宽容：
- 最后一轮仅有 P1 问题、且 P1 可在生产环境被监控/容忍 → 也可以给 PASS
- 但是 P0 永远不放过

## 解析说明

编排脚本使用以下命令检测：

```bash
tail -n 50 "$report" | grep -qE "^STATUS: PASS$"
tail -n 50 "$report" | grep -qE "^STATUS: NEEDS_REVISION$"
```

因此：
- 必须在文件**末尾 50 行内**出现
- 必须**独占一行**
- 大小写敏感，**完全匹配**
- 不要加 `**STATUS: PASS**`、`> STATUS: PASS`、`# STATUS: PASS` 等修饰
